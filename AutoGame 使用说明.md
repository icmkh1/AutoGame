# 使用说明

## 概述

本软件是一个基于 Python 的键鼠自动化工具，支持多种宏功能类型，可通过 JSON 配置文件进行配置。系统支持热键触发、图像匹配、颜色匹配、文字识别等功能。

## 文件结构

```
AutoGame/
├── data/
    │── macrofile         # 宏配置文件目录
    │
    └── target_image      # 图像识别功能 图像放置目录
```

## 配置文件格式

配置文件采用 JSON 数组格式，包含一个基本信息配置项和多个宏功能配置项。

### 基本信息配置

```json
{
    "备注": "基本信息",
    "按键更改": "",
    "坐标更改": "(2560,1440)",
    "鼠标图标更改": "是"
}
```

| 字段     | 类型     | 说明                 |
| ------ | ------ | ------------------ |
| 备注     | string | 配置说明               |
| 按键更改   | string | 预留字段（暂未实现功能）       |
| 坐标更改   | string | 预留字段（暂未实现功能）       |
| 鼠标图标更改 | string | 是否更改鼠标图标，"是" 或 "否" |

### 支持的功能类型

支持以下 11 种宏功能类型：

| 功能类型 | 说明                  |
| ---- | ------------------- |
| 连击   | 按住触发键时持续重复执行宏指令     |
| 固定连击 | 执行指定次数的连击，可配置后置指令   |
| 宏    | 单次执行宏指令             |
| 有序宏  | 按住触发键时循环执行多个宏指令序列   |
| 跟随   | 按键按下时按下指定键，弹起时弹起指定键 |
| 组合   | 根据辅助键状态执行不同分支       |
| 映射   | 根据辅助键状态映射到不同按键      |
| 图像匹配 | 根据图像匹配结果执行不同分支      |
| 颜色匹配 | 根据颜色匹配结果执行不同分支      |
| 文字识别 | 根据文字识别结果执行不同分支      |

***

## 功能类型详解

### 1. 连击

按住触发键时，以指定频率持续执行宏指令。

```json
{
    "功能类型": "连击",
    "触发键": "P",
    "每秒次数": "30",
    "宏指令": "D"
}
```

| 字段   | 类型     | 必填 | 说明           |
| ---- | ------ | -- | ------------ |
| 功能类型 | string | 是  | 固定值 "连击"     |
| 触发键  | string | 是  | 触发该宏的按键      |
| 每秒次数 | string | 是  | 每秒执行次数       |
| 宏指令  | string | 是  | 要执行的宏指令（可多个） |

### 2. 固定连击

执行指定次数的连击，可配置连击间隔和后置指令。

```json
{
    "功能类型": "固定连击",
    "触发键": "P",
    "名称": "测试",
    "连击次数": "10",
    "连击间隔": "0.1",
    "宏指令": "D",
    "后置指令": "A"
}
```

| 字段   | 类型     | 必填     | 说明         |
| ---- | ------ | ------ | ---------- |
| 功能类型 | string | 是      | 固定值 "固定连击" |
| 触发键  | string | 至少存在一种 | 触发该宏的按键    |
| 名称   | string | 至少存在一种 | 可供其他功能调用   |
| 连击次数 | string | 是      | 连击执行次数     |
| 连击间隔 | string | 是      | 每次连击间隔（秒）  |
| 宏指令  | string | 是      | 要执行的宏指令    |
| 后置指令 | string | 否      | 连击完成后执行的指令 |

### 3. 宏

单次触发执行宏指令。

```json
{
    "功能类型": "宏",
    "触发键": "P",
    "名称": "测试",
    "宏指令": "D"
}
```

| 字段   | 类型     | 必填     | 说明       |
| ---- | ------ | ------ | -------- |
| 功能类型 | string | 是      | 固定值 "宏"  |
| 触发键  | string | 至少存在一种 | 触发该宏的按键  |
| 名称   | string | 至少存在一种 | 可供其他功能调用 |
| 宏指令  | string | 是      | 要执行的宏指令  |

### 4. 有序宏

循环执行多个宏指令，可配置后置指令。

```json
{
    "功能类型": "有序宏",
    "触发键": "P",
    "宏指令": "1,2,3,4",
    "后置指令": "延迟 0.1,A,延迟 0.1,A,延迟 0.1"
}
```

| 字段   | 类型     | 必填 | 说明           |
| ---- | ------ | -- | ------------ |
| 功能类型 | string | 是  | 固定值 "有序宏"    |
| 触发键  | string | 是  | 触发该宏的按键      |
| 宏指令  | string | 是  | 逗号分隔的宏指令序列   |
| 后置指令 | string | 否  | 每个指令执行后执行的指令 |

### 5. 跟随

按键按下时按下指定键，弹起时弹起指定键。

```json
{
    "功能类型": "跟随",
    "触发键": "P",
    "宏指令": "D"
}
```

| 字段   | 类型     | 必填 | 说明          |
| ---- | ------ | -- | ----------- |
| 功能类型 | string | 是  | 固定值 "跟随"    |
| 触发键  | string | 是  | 触发该宏的按键     |
| 宏指令  | string | 是  | 要跟随的按键（可多个） |

### 6. 组合

根据辅助键状态执行不同分支或执行统一宏指令。

```json
{
    "功能类型": "组合",
    "触发键": "P",
    "辅助1": "Numpad1",
    "辅助2": "Numpad3",
    "宏指令": "弹起 辅助,延迟 0.1,辅助,!辅助"
}
```

或使用分支模式：

```json
{
    "功能类型": "组合",
    "触发键": "P",
    "辅助1": "Numpad1",
    "辅助2": "Numpad3",
    "分支1": "按下 MLeft 2307 957,移动 2237 957 0.18,弹起 MLeft",
    "分支2": "按下 MLeft 2307 957,移动 2377 957 0.18,弹起 MLeft"
}
```

| 字段   | 类型     | 必填  | 说明          |
| ---- | ------ | --- | ----------- |
| 功能类型 | string | 是   | 固定值 "组合"    |
| 触发键  | string | 是   | 触发该宏的按键     |
| 辅助1  | string | 是   | 第一个辅助键      |
| 辅助2  | string | 是   | 第二个辅助键      |
| 宏指令  | string | 二选一 | 使用宏指令模式时的指令 |
| 分支1  | string | 二选一 | 辅助1按下时执行的指令 |
| 分支2  | string | 二选一 | 辅助2按下时执行的指令 |

**宏指令中可使用的变量：**

- `辅助` - 当前按下的辅助键
- `!辅助` - 未按下的辅助键

### 7. 映射

根据辅助键状态映射到不同按键。

```json
{
    "功能类型": "映射",
    "触发键": "P",
    "辅助1": "Numpad1",
    "辅助2": "Numpad3",
    "映射1": "Numpad7",
    "映射2": "Numpad9",
    "宏指令": "弹起 辅助,延迟 0.1,按下 映射,延迟 0.42,弹起 映射,延迟 0.1,按下 !映射,延迟 0.42,弹起 !映射"
}
```

| 字段   | 类型     | 必填 | 说明        |
| ---- | ------ | -- | --------- |
| 功能类型 | string | 是  | 固定值 "映射"  |
| 触发键  | string | 是  | 触发该宏的按键   |
| 辅助1  | string | 是  | 第一个辅助键    |
| 辅助2  | string | 是  | 第二个辅助键    |
| 映射1  | string | 是  | 辅助1对应的映射键 |
| 映射2  | string | 是  | 辅助2对应的映射键 |
| 宏指令  | string | 是  | 要执行的宏指令   |

**宏指令中可使用的变量：**

- `辅助` - 当前按下的辅助键
- `!辅助` - 未按下的辅助键
- `映射` - 当前辅助键对应的映射键
- `!映射` - 另一辅助键对应的映射键

### 8. 图像匹配

在指定区域内查找目标图像，根据匹配结果执行不同分支。

```json
{
    "功能类型": "图像匹配",
    "图像名称": "1.png",
    "匹配范围": "0 0 100 100",
    "相似度": "0.8",
    "触发键": "P",
    "名称": "测试",
    "分支Y": "按下左键,延迟 0.02,弹起左键",
    "分支N": "P"
}
```

| 字段   | 类型     | 必填     | 说明                            |
| ---- | ------ | ------ | ----------------------------- |
| 功能类型 | string | 是      | 固定值 "图像匹配"                    |
| 图像名称 | string | 是      | 目标图像文件名（需放在指定target\_image目录） |
| 匹配范围 | string | 否      | 搜索区域，格式为 "x1 y1 x2 y2"，默认为全屏  |
| 相似度  | string | 否      | 匹配相似度阈值（0-1），默认 0.8           |
| 触发键  | string | 至少存在一种 | 触发该宏的按键                       |
| 名称   | string | 至少存在一种 | 可供其他功能调用                      |
| 分支N  | string | 否      | 匹配失败时执行的指令                    |

### 9. 颜色匹配

检查指定坐标的颜色是否匹配，根据结果执行不同分支。

```json
{
    "功能类型": "颜色匹配",
    "颜色": "#1F2430,#FFCC66",
    "坐标": "2133 468,26 1387",
    "模式": "all",
    "相似度": "0.8",
    "触发键": "P",
    "名称": "测试",
    "分支Y": "F",
    "分支N": "C"
}
```

| 字段   | 类型     | 必填     | 说明                                    |
| ---- | ------ | ------ | ------------------------------------- |
| 功能类型 | string | 是      | 固定值 "颜色匹配"                            |
| 颜色   | string | 是      | 目标颜色，支持多个颜色（逗号分隔），格式为十六进制颜色码          |
| 坐标   | string | 是      | 要检查的坐标，支持多个坐标（逗号分隔），格式为 "x y"         |
| 模式   | string | 否      | "all"（所有坐标匹配）或 "any"（任一坐标匹配），默认 "all" |
| 相似度  | string | 否      | 颜色相似度阈值（0-1），默认 0.8                   |
| 触发键  | string | 至少存在一种 | 触发该宏的按键                               |
| 名称   | string | 至少存在一种 | 可供其他功能调用                              |
| 分支Y  | string | 否      | 匹配成功时执行的指令                            |
| 分支N  | string | 否      | 匹配失败时执行的指令                            |

### 10. 文字识别

在指定区域内识别文字，根据识别结果执行不同分支。

```json
{
    "功能类型": "文字识别",
    "识别文本": "hello world",
    "寻找范围": "0 0 100 100",
    "模式": "all",
    "触发键": "P",
    "名称": "测试",
    "分支Y": "F",
    "分支N": "J"
}
```

| 字段   | 类型     | 必填     | 说明                                  |
| ---- | ------ | ------ | ----------------------------------- |
| 功能类型 | string | 是      | 固定值 "文字识别"                          |
| 识别文本 | string | 是      | 要识别的目标文本                            |
| 寻找范围 | string | 否      | 搜索区域，格式为 "x1 y1 x2 y2"，默认为全屏        |
| 模式   | string | 否      | "all"（完全匹配）或 "any"（包含任一字符），默认 "all" |
| 触发键  | string | 至少存在一种 | 触发该宏的按键                             |
| 名称   | string | 至少存在一种 | 可供其他功能调用                            |
| 分支Y  | string | 否      | 识别成功时执行的指令                          |
| 分支N  | string | 否      | 识别失败时执行的指令                          |

## 宏指令语法

### 通过名称调用功能

除了直接执行按键指令外，还可以在宏指令中通过功能模块的 `名称` 字段来调用其他已配置的功能。配置时需要在功能项中添加 `名称` 字段来标识该功能。

以下功能类型支持设置 `名称` 并被其他宏指令调用：

| 支持设置名称 | 支持调用名称 | 都不支持   |
| ------ | ------ | :----- |
| 固定连击   | 连击     | 跟随     |
| 宏      | 固定连击   | <br /> |
| 图像匹配   | 宏      | <br /> |
| 颜色匹配   | 有序宏    | <br /> |
| 文字识别   | 组合     | <br /> |
| <br /> | 映射     | <br /> |
| <br /> | 图像匹配   | <br /> |
| <br /> | 颜色匹配   | <br /> |
| <br /> | 文字识别   | <br /> |

<br />

**名称调用示例：**

```json
[
    {
        "备注": "基本信息",
        "按键更改": "",
        "坐标更改": "(2560,1440)",
        "鼠标图标更改": "是"
    },
    {
        "名称": "连招A",
        "功能类型": "固定连击",
        "连击次数": "5",
        "连击间隔": "0.1",
        "宏指令": "D"
    },
    {
        "名称": "连招B",
        "功能类型": "宏",
        "宏指令": "A,B,C,D"
    },
    {
        "功能类型": "宏",
        "触发键": "P",
        "宏指令": "连招A,延迟 0.5,连招B"
    }
]
```

上例中，当按下 P 键时，会先调用名为 `连招A` 的固定连击功能，执行 5 次连击，间隔 0.1 秒，然后延迟 0.5 秒，最后调用名为 `连招B` 的宏功能，执行 A B C D。

宏指令由多个动作组成，用逗号分隔。每个动作由命令和参数组成。

### 支持的指令类型

| 指令   | 语法                          | 说明                                                    |
| ---- | --------------------------- | ----------------------------------------------------- |
| 单击   | `按键名` 或 `MLeft x y`         | 单击键盘按键或鼠标指定位置                                         |
| 按下   | `按下 按键名` 或 `按下 MLeft x y`   | 按下键盘按键或鼠标指定位置                                         |
| 弹起   | `弹起 按键名` 或 `弹起 MLeft x y`   | 弹起键盘按键或鼠标指定位置                                         |
| 移动   | `移动 x y [duration] [steps]` | 移动鼠标到指定位置，可选参数： duration=用时(秒) steps=步数(整数)，默认 0.2和10 |
| 滚轮   | `滚轮 amount [x y]`           | 滚动鼠标滚轮，amount=正整数或负整数                                 |
| 延迟   | `延迟 seconds`                | 延迟指定时间（秒）                                             |
| 调用功能 | `功能名称`                      | 通过名称调用已配置的功能模块                                        |

### 鼠标按钮名称

| 按钮名称   | 说明    |
| ------ | ----- |
| MLeft  | 鼠标左键  |
| MRight | 鼠标右键  |
| Middle | 鼠标中键  |
| side1  | 鼠标侧键1 |
| side2  | 鼠标侧键2 |

### 指令示例

```
D                           // 单击 D 键
按下 D                       // 按下 D 键
弹起 D                       // 弹起 D 键
MLeft 100 200                // 在坐标 (100, 200) 单击左键
按下 MLeft 100 200           // 在坐标 (100, 200) 按下左键
弹起 MLeft 100 200           // 在坐标 (100, 200) 弹起左键
移动 100 200 0.5             // 用 0.5 秒移动鼠标到 (100, 200)
滚轮 5                       // 向上滚动滚轮 5 格
延迟 0.5                     // 延迟 0.5 秒
```

### 组合指令示例

```
按下 MLeft 2307 957,移动 2237 957 0.18,弹起 MLeft,MLeft
```
