# 使用说明

## 概述

本软件是一个基于 Python 的自动化工具，支持前后台键鼠、图色、文字识别等功能。

## 文件结构

```
AutoGame/
├── data/
    │── macrofile         # 宏配置文件目录
    │
    └── target_image      # 图像识别功能 图像文件放置目录
```

## 配置文件格式

配置文件采用 JSON 数组格式，包含一个基本信息配置项和多个宏功能配置项。

### 基本信息配置

```json
{
    "备注": "基本信息",
    "按键更改": "",
    "坐标更改": "(2560,1440)",
    "窗口标题": "",
    "窗口类名": "",
    "鼠标图标更改": "是"
}
```

| 字段     | 类型     | 说明                     |
| ------ | ------ | ---------------------- |
| 备注     | string | 配置说明                   |
| 按键更改   | string | 预留字段（暂未实现功能）           |
| 坐标更改   | string | 预留字段（暂未实现功能）           |
| 窗口标题   | string | 标题和类名任何一个被设置则此文件进入后台模式 |
| 窗口类名   | string | 标题和类名任何一个被设置则此文件进入后台模式 |
| 鼠标图标更改 | string | 是否更改鼠标图标，"是" 或 "否"     |

### 支持的功能类型

| 功能类型 | 说明                    |
| ---- | --------------------- |
| 连击   | 按住触发键时持续重复执行宏指令       |
| 固定连击 | 执行指定次数的连击，可配置后置指令     |
| 宏    | 单次执行宏指令               |
| 有序宏  | 按住触发键时循环执行多个宏指令序列     |
| 跟随   | 按键按下时按下指定键，弹起时弹起指定键   |
| 组合   | 根据辅助键状态执行不同分支         |
| 映射   | 根据辅助键状态映射到不同按键        |
| 截图   | 截取屏幕截图并保存到指定目录        |
| 追踪   | 根据目标颜色和追踪颜色所在位置执行不同分支 |
| 图像匹配 | 根据图像匹配结果执行不同分支        |
| 颜色匹配 | 根据颜色匹配结果执行不同分支        |
| 文字识别 | 根据文字识别结果执行不同分支        |

## 功能类型详解

### 连击

按住触发键时，以指定频率持续执行宏指令。

```json
{
    "功能类型": "连击",
    "触发键": "P",
    "每秒次数": "30",
    "宏指令": "D"
}
```

| 字段   | 类型     | 必填 | 说明           |
| ---- | ------ | -- | ------------ |
| 功能类型 | string | 是  | 固定值 "连击"     |
| 触发键  | string | 是  | 触发该宏的按键      |
| 每秒次数 | string | 是  | 每秒执行次数       |
| 宏指令  | string | 是  | 要执行的宏指令（可多个） |

### 固定连击

执行指定次数的连击，可配置连击间隔和后置指令。

```json
{
    "功能类型": "固定连击",
    "触发键": "P",
    "名称": "测试",
    "连击次数": "10",
    "连击间隔": "0.1",
    "宏指令": "D",
    "后置指令": "A"
}
```

| 字段   | 类型     | 必填     | 说明         |
| ---- | ------ | ------ | ---------- |
| 功能类型 | string | 是      | 固定值 "固定连击" |
| 触发键  | string | 至少存在一种 | 触发该宏的按键    |
| 名称   | string | 至少存在一种 | 可供其他功能调用   |
| 连击次数 | string | 是      | 连击执行次数     |
| 连击间隔 | string | 是      | 每次连击间隔（秒）  |
| 宏指令  | string | 是      | 要执行的宏指令    |
| 后置指令 | string | 否      | 连击完成后执行的指令 |

### 宏

单次触发执行宏指令。

```json
{
    "功能类型": "宏",
    "触发键": "P",
    "名称": "测试",
    "宏指令": "D"
}
```

| 字段   | 类型     | 必填     | 说明       |
| ---- | ------ | ------ | -------- |
| 功能类型 | string | 是      | 固定值 "宏"  |
| 触发键  | string | 至少存在一种 | 触发该宏的按键  |
| 名称   | string | 至少存在一种 | 可供其他功能调用 |
| 宏指令  | string | 是      | 要执行的宏指令  |

### 有序宏

循环执行多个宏指令，可配置后置指令。

```json
{
    "功能类型": "有序宏",
    "触发键": "P",
    "宏指令": "1,2,3,4",
    "后置指令": "延迟 0.1,A,延迟 0.1,A,延迟 0.1"
}
```

| 字段   | 类型     | 必填 | 说明           |
| ---- | ------ | -- | ------------ |
| 功能类型 | string | 是  | 固定值 "有序宏"    |
| 触发键  | string | 是  | 触发该宏的按键      |
| 宏指令  | string | 是  | 逗号分隔的宏指令序列   |
| 后置指令 | string | 否  | 每个指令执行后执行的指令 |

### 跟随

按键按下时按下指定键，弹起时弹起指定键。

```json
{
    "功能类型": "跟随",
    "触发键": "P",
    "宏指令": "D"
}
```

| 字段   | 类型     | 必填 | 说明                   |
| ---- | ------ | -- | -------------------- |
| 功能类型 | string | 是  | 固定值 "跟随"             |
| 触发键  | string | 是  | 触发该宏的按键              |
| 宏指令  | string | 是  | 要跟随的按键（可多个，且只能为单击指令） |

### 组合

根据辅助键状态执行不同分支或执行统一宏指令。

```json
{
    "功能类型": "组合",
    "触发键": "P",
    "辅助1": "Numpad1",
    "辅助2": "Numpad3",
    "宏指令": "弹起 辅助,延迟 0.1,辅助,!辅助"
}
```

或使用分支模式：

```json
{
    "功能类型": "组合",
    "触发键": "P",
    "辅助1": "Numpad1",
    "辅助2": "Numpad3",
    "分支1": "按下 MLeft 2307 957,移动 2237 957 0.18,弹起 MLeft",
    "分支2": "按下 MLeft 2307 957,移动 2377 957 0.18,弹起 MLeft"
}
```

| 字段   | 类型     | 必填  | 说明          |
| ---- | ------ | --- | ----------- |
| 功能类型 | string | 是   | 固定值 "组合"    |
| 触发键  | string | 是   | 触发该宏的按键     |
| 辅助1  | string | 是   | 第一个辅助键      |
| 辅助2  | string | 是   | 第二个辅助键      |
| 宏指令  | string | 二选一 | 使用宏指令模式时的指令 |
| 分支1  | string | 二选一 | 辅助1按下时执行的指令 |
| 分支2  | string | 二选一 | 辅助2按下时执行的指令 |

**宏指令中可使用的变量：**

- `辅助` - 当前按下的辅助键
- `!辅助` - 未按下的辅助键

### 映射

根据辅助键状态映射到不同按键。

```json
{
    "功能类型": "映射",
    "触发键": "P",
    "辅助1": "Numpad1",
    "辅助2": "Numpad3",
    "映射1": "Numpad7",
    "映射2": "Numpad9",
    "宏指令": "弹起 辅助,延迟 0.1,按下 映射,延迟 0.42,弹起 映射,延迟 0.1,按下 !映射,延迟 0.42,弹起 !映射"
}
```

| 字段   | 类型     | 必填 | 说明        |
| ---- | ------ | -- | --------- |
| 功能类型 | string | 是  | 固定值 "映射"  |
| 触发键  | string | 是  | 触发该宏的按键   |
| 辅助1  | string | 是  | 第一个辅助键    |
| 辅助2  | string | 是  | 第二个辅助键    |
| 映射1  | string | 是  | 辅助1对应的映射键 |
| 映射2  | string | 是  | 辅助2对应的映射键 |
| 宏指令  | string | 是  | 要执行的宏指令   |

**宏指令中可使用的变量：**

- `辅助` - 当前按下的辅助键
- `!辅助` - 未按下的辅助键
- `映射` - 当前辅助键对应的映射键
- `!映射` - 另一辅助键对应的映射键

### 截图

在指定区域或全屏区域截图。

```json
{
    "功能类型": "截图",
    "触发键": "P",
    "名称": "测试",
    "文件名称": "screenshot",
    "截图范围": "100 100 1000 1000"
}
```

| 字段   | 类型     | 必填     | 说明                                                |
| ---- | ------ | ------ | ------------------------------------------------- |
| 功能类型 | string | 是      | 固定值 "截图"                                          |
| 触发键  | string | 至少存在一种 | 触发该宏的按键                                           |
| 名称   | string | 至少存在一种 | 可供其他功能调用                                          |
| 文件名称 | string | 否      | 截图保存在 `data\target_image` 文件夹下，默认名称为 `screenshot` |
| 截图范围 | string | 否      | 格式为 "x1 y1 x2 y2"，默认为全屏                           |

### 追踪

根据目标颜色和追踪颜色所在位置执行不同分支。

```json
{
    "功能类型": "追踪",
    "触发键": "P",
    "名称": "测试",
    "追踪补偿": "20",
    "目标颜色": "#FEF7A5",
    "追踪颜色": "#2BCAAD",
    "匹配范围": "410 49 875 50",
    "大于分支": "D",
    "小于分支": "A",
    "后置指令": "测试1"
}
```

上例中，如果 目标颜色（单点颜色） 在 追踪颜色（横条颜色） 的右边，执行 大于分支 ，否则执行 小于分支，同理可用于上下匹配。


| 字段   | 类型     | 必填     | 说明                           |
| ---- | ------ | ------ | ---------------------------- |
| 功能类型 | string | 是      | 固定值 "追踪"                     |
| 触发键  | string | 至少存在一种 | 触发该宏的按键                      |
| 名称   | string | 至少存在一种 | 可供其他功能调用                     |
| 追踪补偿 | string | 是      | 追踪颜色的 `头部+补偿值` `尾部-补偿值`                     |
| 目标颜色 | string | 是      | 目标颜色（十六进制格式），为单点颜色                 |
| 追踪颜色 | string | 是      | 追踪颜色（十六进制格式），为横条或竖条颜色                 |
| 匹配范围 | string | 否      | 搜索区域，格式为 "x1 y1 x2 y2"，只能高度或宽度为1，长度不限  |
| 大于分支 | string | 是      | 匹配成功且目标颜色大于追踪颜色时执行的指令        |
| 小于分支 | string | 是      | 匹配成功且目标颜色小于追踪颜色时执行的指令        |
| 后置指令 | string | 是      | 匹配完成后执行的指令                   |

## 后台模式

### 进入后台模式

```json
{
    "备注": "基本信息",
    "按键更改": "",
    "坐标更改": "(2560,1440)",
    "窗口标题": "MuMuNxDevice",
    "窗口类名": "Qt5156QWindowIcon",
    "鼠标图标更改": "是"
}
```

如上例，在基础信息设置中配置 `窗口标题` 或 `窗口类名`，整个文件的 `所有功能` 都自动进入后台模式

### 后台模式独有字段

```json
{
    "功能类型": "宏",
    "触发键": "P",
    "键鼠模式": "post",
    "宏指令": "MLeft 738 59"
}
```

| 字段   | 类型     | 必填 | 说明                                                         |
| ---- | ------ | -- | ---------------------------------------------------------- |
| 键鼠模式 | string | 否  | 所有功能都可以选填，类型有: 'send', 'post', 'global' 三种模式，默认为 'send' 模式 |

`send` 模式为同步模式，向窗口发送消息，等待窗口执行，且等待返回值

`post` 模式为异步模式，向窗口发送消息，不等待窗口执行，不等待返回值

`global` 模式为全局模式，使用前台键鼠操作，上述模式失效时建议使用此模式

### 后台模式注意事项

坐标：在后台模式下，坐标是相对于窗口的，不再是相对于屏幕的，global 模式也是相对于窗口的坐标，
例如：x=0, y=0，这在前台模式下表示 `屏幕` 的最左上角，但是在后台模式下，它表示 `窗口` 的最左上角。

图色文字匹配：某些窗口在最小化到任务栏后，是有很大可能无法获取到窗口的图像的，所以建议窗口在前台打开。

窗口级别：如果 '窗口标题' 或 '窗口类名' 为一级窗口，是有可能操作不了二级、三级窗口的。
坐标也是会受到窗口层级影响的，建议先使用 `截图功能` 截取窗口图像，然后使用windows的 `画图` 工具来获取坐标。

## 宏指令语法

### 通过名称调用功能

除了直接执行按键指令外，还可以在宏指令中通过功能模块的 `名称` 字段来调用其他已配置的功能。配置时需要在功能项中添加 `名称` 字段来标识该功能。

以下功能类型支持设置 `名称` 并被其他宏指令调用：

| 支持设置名称 | 支持调用名称 | 都不支持   |
| ------ | ------ | :----- |
| 固定连击   | 连击     | 跟随     |
| 宏      | 固定连击   | <br /> |
| 图像匹配   | 宏      | <br /> |
| 颜色匹配   | 有序宏    | <br /> |
| 文字识别   | 组合     | <br /> |
| <br /> | 映射     | <br /> |
| <br /> | 图像匹配   | <br /> |
| <br /> | 颜色匹配   | <br /> |
| <br /> | 文字识别   | <br /> |

<br />

**名称调用示例：**

```json
[
    {
        "备注": "基本信息",
        "按键更改": "",
        "坐标更改": "(2560,1440)",
        "鼠标图标更改": "是"
    },
    {
        "名称": "连招A",
        "功能类型": "固定连击",
        "连击次数": "5",
        "连击间隔": "0.1",
        "宏指令": "D"
    },
    {
        "名称": "连招B",
        "功能类型": "宏",
        "宏指令": "A,B,C,D"
    },
    {
        "功能类型": "宏",
        "触发键": "P",
        "宏指令": "连招A,延迟 0.5,连招B"
    }
]
```

上例中，当按下 P 键时，会先调用名为 `连招A` 的固定连击功能，执行 5 次连击，间隔 0.1 秒，然后延迟 0.5 秒，最后调用名为 `连招B` 的宏功能，执行 A B C D。

宏指令由多个动作组成，用逗号分隔。每个动作由命令和参数组成。

### 支持的指令类型

| 指令   | 语法                          | 说明                                                    |
| ---- | --------------------------- | ----------------------------------------------------- |
| 单击   | `按键名` 或 `MLeft x y`         | 单击键盘按键或鼠标指定位置                                         |
| 按下   | `按下 按键名` 或 `按下 MLeft x y`   | 按下键盘按键或鼠标指定位置                                         |
| 弹起   | `弹起 按键名` 或 `弹起 MLeft x y`   | 弹起键盘按键或鼠标指定位置                                         |
| 移动   | `移动 x y [duration] [steps]` | 移动鼠标到指定位置，可选参数： duration=用时(秒) steps=步数(整数)，默认 0.2和10 |
| 滚轮   | `滚轮 amount [x y]`           | 滚动鼠标滚轮，amount=正整数或负整数                                 |
| 延迟   | `延迟 seconds`                | 延迟指定时间（秒）                                             |
| 调用功能 | `功能名称`                      | 通过名称调用已配置的功能模块                                        |

### 鼠标按钮名称

| 按钮名称   | 说明    |
| ------ | ----- |
| MLeft  | 鼠标左键  |
| MRight | 鼠标右键  |
| Middle | 鼠标中键  |
| MSide1  | 鼠标侧键1 |
| MSide2  | 鼠标侧键2 |

### 指令示例

```
D                           // 单击 D 键
按下 D                       // 按下 D 键
弹起 D                       // 弹起 D 键
MLeft 100 200                // 在坐标 (100, 200) 单击左键
按下 MLeft 100 200           // 在坐标 (100, 200) 按下左键
弹起 MLeft 100 200           // 在坐标 (100, 200) 弹起左键
移动 100 200 0.5             // 用 0.5 秒移动鼠标到 (100, 200)
滚轮 5                       // 向上滚动滚轮 5 格
延迟 0.5                     // 延迟 0.5 秒
```

### 组合指令示例

```
按下 MLeft 2307 957,移动 2237 957 0.18,弹起 MLeft,MLeft
```
